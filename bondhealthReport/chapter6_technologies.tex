\section{Frontend Technologies}

Frontend development utilizes HTML5 for semantic markup, Tailwind CSS for responsive utility-first styling, and Font Awesome for icons. Bootstrap 5 provides additional form and modal components for hospital registration pages. The responsive design ensures optimal viewing on desktops, tablets, and mobile devices.

\subsection{Key Frontend Features}
\begin{itemize}
    \item Semantic HTML5 with form validation attributes
    \item Tailwind CSS utility classes for custom styling (custom colors: cyan-bg, cyan-text)
    \item Bootstrap 5 grids and components for forms
    \item Font Awesome icons (6400+) for navigation and actions
    \item Client-side data persistence with localStorage
    \item Custom CSS animations (fade-in, slide-in, hover effects)
\end{itemize}

\section{Backend Technologies}

\subsection{Node.js and Express.js}
Node.js v18.17.0 (LTS) runs the application server on port 3005 using an event-driven, non-blocking I/O model. Express.js v4.18.2 provides the core framework with routing, middleware, and error handling for 30+ API endpoints.

Key usage:
\begin{itemize}
    \item Event loop for efficient concurrent request handling
    \item File system module (fs) for serving HTML templates
    \item Middleware for authentication, authorization, and logging
    \item Error handling with try-catch blocks
\end{itemize}

\subsection{PostgreSQL and Neon Serverless}
PostgreSQL v15 provides the relational database with 15+ tables maintaining ACID compliance and data integrity. Neon serverless platform enables automatic scaling and connection pooling without infrastructure management.

Database features:
\begin{itemize}
    \item Foreign key relationships for data consistency
    \item UUID primary keys for distributed systems
    \item Indexing for query performance
    \item Point-in-time recovery for data protection
    \item SSL connections for encrypted communication
\end{itemize}

\section{Authentication \& Security}

\subsection{JWT and bcryptjs}
JSON Web Tokens (jsonwebtoken v9.0.3) handle stateless authentication with 7-day expiration. bcryptjs v3.0.3 provides password hashing with 10 salt rounds against rainbow table attacks.

\subsection{Security Implementation}
\begin{itemize}
    \item Role-based access control (RBAC) middleware for four distinct roles
    \item HTTP-only cookies for token storage (prevents XSS)
    \item Parameterized queries to prevent SQL injection
    \item Input validation and sanitization
    \item Secure password comparison with timing-safe functions
\end{itemize}

\section{Development Tools}

Version control with Git and GitHub enables collaborative development using feature branches and pull requests. VS Code with extensions (LaTeX Workshop, Thunder Client, Prettier) streamlines development. npm v9.6.7 manages all project dependencies defined in package.json, including:

\begin{itemize}
    \item Production: express, pg, jsonwebtoken, bcryptjs, cookie-parser, dotenv
    \item Development: nodemon (auto-reload), concurrently (parallel process execution)
\end{itemize}

\section{Project Structure}

BondHealth follows a modular architecture with clear separation of concerns:

\begin{itemize}
    \item \textbf{src/} - Main source code directory
    \begin{itemize}
        \item \textbf{db/} - Database configuration files
        \begin{itemize}
            \item \texttt{config.js} - Database connection
            \item \texttt{init.js} - Database initialization
            \item \texttt{init.sql} - SQL schema (15+ tables)
            \item \texttt{seed.js} - Seed data
            \item \texttt{seed-reviews.js} - Review data seeding
        \end{itemize}
        \item \texttt{Patient.js} - Patient dashboard module
        \item \texttt{Doctor.js} - Doctor dashboard module
        \item \texttt{admin.js} - Admin dashboard module
        \item \texttt{labs.js} - Lab technician module
        \item \texttt{signin.js} - Authentication server
        \item \texttt{home.js} - Main server with 30+ API endpoints
        \item \texttt{Hospital.js} - Hospital management module
        \item \texttt{HospitalRegistration.js} - Hospital registration
        \item \texttt{signup\_patient.js} - Patient registration
        \item \texttt{clone.js} - Appointment booking demo
    \end{itemize}
    \item \texttt{.env} - Environment variables
    \item \texttt{.gitignore} - Git ignore rules
    \item \texttt{package.json} - Dependencies and scripts
    \item \texttt{README.md} - Project documentation
\end{itemize}

\section{Deployment Configuration}

The application uses environment variables (dotenv v17.3.1) for configuration:
\begin{itemize}
    \item \textbf{Development}: Local/Neon DB with debug logging
    \item \textbf{Production}: Neon serverless PostgreSQL with SSL, connection pooling, and optimized security settings
\end{itemize}

\section{Advantages of Technology Stack}

\subsection{Performance}
Non-blocking I/O in Node.js handles multiple concurrent connections efficiently. Tailwind CSS's production build purges unused styles. Database connection pooling optimizes query execution.

\subsection{Scalability}
Serverless PostgreSQL with Neon automatically scales compute resources. Modular architecture supports horizontal scaling. Stateless JWT authentication eliminates session bottlenecks.

\subsection{Security}
Industry-standard JWT authentication with role-based access control. bcryptjs password hashing with adequate salt rounds. Parameterized queries prevent SQL injection. HTTP-only cookies mitigate XSS attacks.

\subsection{Developer Experience}
Express.js minimal framework reduces boilerplate. Tailwind CSS utility-first approach accelerates UI development. Git version control with branching strategy supports team collaboration. Clear project structure aids code maintainability.

The combination of these technologies provides a solid foundation for a responsive, secure, and scalable healthcare platform serving patients, doctors, administrators, and lab technicians.
