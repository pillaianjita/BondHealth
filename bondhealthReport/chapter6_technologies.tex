\section{Frontend Technologies}

This section describes the frontend technologies used to create BondHealth's responsive and intuitive user interfaces across all dashboards.

\begin{itemize}
    \item \textbf{HTML5:} Provides semantic structure with elements like \texttt{<header>}, \texttt{<nav>}, and \texttt{<section>}. Includes built-in form validation, local storage for session persistence, and file upload capabilities for medical reports.
    
    \item \textbf{Tailwind CSS:} A utility-first CSS framework used extensively across all dashboards (Patient.js, Doctor.js, admin.js). Enables rapid UI development with pre-built classes for responsive design, custom theming with cyan-bg and cyan-text classes, and built-in animations for smooth interactions.
    
    \item \textbf{Bootstrap 5:} Employed in hospital management pages (Hospital.js and HospitalRegistration.js) for rapid form development, providing styled form controls, responsive grid layouts, and modal components.
    
    \item \textbf{Font Awesome:} Provides over 6400 scalable vector icons used throughout the application for navigation menu items, action buttons, status indicators, profile avatars, and medical symbols.
\end{itemize}

\section{Backend Technologies}

This section covers the server-side technologies that power BondHealth's business logic, API endpoints, and data processing capabilities.

\subsection{Node.js and Express.js}
Node.js v18.17.0 (LTS) powers the application server on port 3005 using an event-driven, non-blocking I/O model. Express.js v4.18.2 provides the core framework with routing, middleware, and error handling for over 30 API endpoints across all dashboards. Key features include async/await for database operations, modular architecture with separate files for each user role, and file system module for serving HTML templates.

\subsection{PostgreSQL and Neon Serverless}
PostgreSQL v15 serves as the relational database with over 15 tables maintaining ACID compliance and data integrity. Neon serverless platform enables automatic scaling and connection pooling without infrastructure management. The database schema includes tables for users, patients, doctors, appointments, lab reports, prescriptions, and medicines, with foreign key relationships ensuring data consistency and UUID primary keys for enhanced security.

\section{Authentication \& Security}

This section details the security technologies and implementations that protect patient data and ensure proper access control.

\subsection{JWT and bcryptjs}
JSON Web Tokens (jsonwebtoken v9.0.3) handle stateless authentication with 7-day expiration, storing user role information for role-based access control. bcryptjs v3.0.3 provides password hashing with 10 salt rounds, protecting against rainbow table attacks.

\subsection{Security Implementation}
BondHealth implements multiple layers of security to protect sensitive healthcare data. Role-based access control (RBAC) middleware ensures that users can only access features appropriate for their role—whether patient, doctor, admin, or lab technician. All JWT tokens are stored in HTTP-only cookies to prevent XSS attacks, while parameterized queries throughout the codebase prevent SQL injection vulnerabilities. Input validation and sanitization are performed on both client and server sides to ensure data integrity. SSL encryption secures all database connections, and environment variables keep sensitive configuration information like database URLs and JWT secrets out of the source code.

\section{Development Tools}

This section describes the tools and utilities used during BondHealth's development lifecycle.

\begin{itemize}
    \item \textbf{Visual Studio Code:} Primary IDE with extensions for LaTeX Workshop (documentation), Prettier (code formatting), ESLint (linting), GitLens (version control), Tailwind CSS IntelliSense, and Thunder Client (API testing).
    
    \item \textbf{Git and GitHub:} Version control system enabling collaborative development with feature branches, pull requests for code review, issues for task tracking, and GitHub Actions for continuous integration.
    
    \item \textbf{NPM (Node Package Manager):} Manages project dependencies including production packages (express, pg, jsonwebtoken, bcryptjs, cookie-parser, dotenv) and development packages (nodemon for auto-reload, concurrently for parallel process execution).
\end{itemize}

\section{Advantages of Technology Stack}

This section summarizes the key benefits of the chosen technology stack in terms of performance, scalability, maintainability, and security.

\begin{itemize}
    \item \textbf{Performance:} Node.js non-blocking I/O handles multiple concurrent requests efficiently. PostgreSQL indexing optimizes query performance. Connection pooling reduces database connection overhead. Lazy loading of dashboard components improves initial load time.
    
    \item \textbf{Scalability:} Horizontal scaling with multiple Node.js instances behind load balancer. Database connection pooling for high concurrency. Serverless database automatically scales compute resources. Stateless JWT authentication enables easy horizontal scaling.
    
    \item \textbf{Maintainability:} Modular code structure with separate files for each user role. Comprehensive documentation. Version control with Git for change tracking. Consistent coding style with ESLint and Prettier. Environment-based configuration for different deployment scenarios.
    
    \item \textbf{Security:} Password hashing with bcrypt. JWT for secure authentication with HTTP-only cookies. SQL injection prevention with parameterized queries. Role-based access control for all API endpoints. Environment variables for sensitive configuration.
\end{itemize}

\section{Conclusion}
The technology stack chosen for BondHealth provides a solid foundation for building a secure, scalable, and maintainable healthcare management system. Node.js/Express delivers a fast, event-driven backend, PostgreSQL with Neon offers reliable data persistence with serverless scaling, JWT with bcrypt ensures secure authentication, and Tailwind CSS with Bootstrap enables rapid, responsive UI development. The combination of these technologies has proven effective in delivering a responsive and user-friendly healthcare platform that meets the needs of all stakeholders.
