\section{Frontend Technologies}

\subsection{HTML5}
HTML5 provides the structural foundation for all web pages in BondHealth. It offers semantic elements that improve accessibility and SEO. Key HTML5 features utilized include:

\begin{itemize}
    \item \textbf{Semantic tags}: \texttt{<header>}, \texttt{<nav>}, \texttt{<section>}, \texttt{<article>}, \texttt{<footer>} for better document structure
    \item \textbf{Form validation}: Built-in validation attributes like \texttt{required}, \texttt{pattern}, and \texttt{type="email"} for enhanced user experience
    \item \textbf{Local storage}: Client-side data persistence for saving user preferences and session state
    \item \textbf{Responsive images}: \texttt{<picture>} element and \texttt{srcset} attributes for optimal image delivery across devices
\end{itemize}

\subsection{Tailwind CSS}
Tailwind CSS v3.3.0 is a utility-first CSS framework that enables rapid UI development. Its advantages in BondHealth include:

\begin{itemize}
    \item \textbf{Utility-first approach}: Pre-built classes for margins, padding, colors, flexbox, and grid systems
    \item \textbf{Responsive design}: Built-in breakpoints (sm, md, lg, xl, 2xl) for mobile, tablet, and desktop views
    \item \textbf{Customization}: Easy theming with the \texttt{tailwind.config.js} file for brand colors and fonts
    \item \textbf{Performance}: Unused styles are purged in production, reducing CSS bundle size by approximately 70\%
\end{itemize}

\section{Backend Technologies}

\subsection{Node.js}
Node.js v18.17.0 (LTS) is a JavaScript runtime built on Chrome's V8 engine. It enables server-side JavaScript execution and provides an event-driven, non-blocking I/O model. Benefits in BondHealth include:

\begin{itemize}
    \item \textbf{Single language}: JavaScript used for both frontend and backend, reducing context switching
    \item \textbf{NPM ecosystem}: Access to over 1.3 million open-source packages
    \item \textbf{Scalability}: Handles multiple concurrent connections efficiently with event loop architecture
    \item \textbf{Performance}: Fast execution for real-time applications with asynchronous I/O
\end{itemize}

\subsection{Express.js}
Express.js v4.18.2 is a minimal and flexible Node.js web application framework. Key features used:

\begin{itemize}
    \item \textbf{Routing}: Handling HTTP requests for different endpoints (\texttt{GET}, \texttt{POST}, \texttt{PUT}, \texttt{DELETE})
    \item \textbf{Middleware}: Parsing JSON, handling CORS, logging requests, error handling
    \item \textbf{Template rendering}: Serving dynamic HTML content through functions
    \item \textbf{Error handling}: Centralized error management with custom error handlers
\end{itemize}

Example Express.js route handler from BondHealth's appointment system:

\begin{lstlisting}[language=JavaScript, caption=Express.js Route Handler from BondHealth, basicstyle=\ttfamily\small, breaklines=true]
// Get today's appointments for doctor dashboard
app.get('/api/appointments/today', async (req, res) => {
  try {
    const doctorId = req.user.doctor_id;
    const result = await query(
      `SELECT a.*, p.full_name as patient_name, p.patient_uuid
       FROM appointments a
       JOIN patients p ON a.patient_id = p.patient_id
       WHERE a.doctor_id = $1 
       AND a.appointment_date = CURRENT_DATE
       ORDER BY a.appointment_time`,
      [doctorId]
    );
    res.json(result.rows);
  } catch (error) {
    console.error('Error fetching appointments:', error);
    res.status(500).json({ error: 'Failed to fetch appointments' });
  }
});

// Book new appointment
app.post('/api/appointments', async (req, res) => {
  try {
    const { doctor_id, appointment_date, appointment_time, reason, type } = req.body;
    const patient_id = req.user.patient_id;
    
    const result = await query(
      `INSERT INTO appointments 
       (patient_id, doctor_id, appointment_date, appointment_time, reason, appointment_type, status)
       VALUES ($1, $2, $3, $4, $5, $6, 'confirmed')
       RETURNING *`,
      [patient_id, doctor_id, appointment_date, appointment_time, reason, type]
    );
    
    res.status(201).json(result.rows[0]);
  } catch (error) {
    console.error('Error booking appointment:', error);
    res.status(500).json({ error: 'Failed to book appointment' });
  }
});
\end{lstlisting}

\section{Database Technologies}

\subsection{PostgreSQL}
PostgreSQL v15 is a powerful, open-source object-relational database system. Its features utilized in BondHealth include:

\begin{itemize}
    \item \textbf{ACID compliance}: Ensures data integrity and reliability for medical records
    \item \textbf{JSON support}: Store and query JSON data for flexible schemas in lab reports
    \item \textbf{Foreign keys}: Maintain referential integrity between tables
    \item \textbf{Indexes}: Optimize query performance for large datasets
    \item \textbf{Transactions}: Group multiple operations into atomic units for data consistency
\end{itemize}

\subsection{Neon Database Serverless}
Neon is a serverless Postgres platform that separates storage and compute for automatic scaling. Advantages include:

\begin{itemize}
    \item \textbf{Serverless architecture}: No infrastructure management, automatic backups
    \item \textbf{Autoscaling}: Handles varying workloads automatically
    \item \textbf{Branching}: Create database branches for development and testing
    \item \textbf{Point-in-time recovery}: Restore data to any point in time
    \item \textbf{Connection pooling}: Built-in pooling for handling many concurrent connections
\end{itemize}

The connection to Neon PostgreSQL is established using:

\begin{lstlisting}[language=JavaScript, caption=Database Configuration in BondHealth, basicstyle=\ttfamily\small, breaklines=true]
const { Pool } = require('@neondatabase/serverless');

const pool = new Pool({
  connectionString: process.env.DATABASE_URL,
  ssl: {
    require: true,
  },
});

const query = async (text, params) => {
  const start = Date.now();
  try {
    const res = await pool.query(text, params);
    const duration = Date.now() - start;
    console.log('Executed query', { text, duration, rows: res.rowCount });
    return res;
  } catch (error) {
    console.error('Query error:', error);
    throw error;
  }
};

module.exports = { pool, query };
\end{lstlisting}

\section{Supporting Technologies}

\subsection{JSON Web Tokens (JWT)}
JWT is used for authentication and authorization:

\begin{itemize}
    \item \textbf{Stateless authentication}: No server-side session storage required
    \item \textbf{Role-based access}: Tokens contain user role information
    \item \textbf{Secure transmission}: Signed tokens prevent tampering
    \item \textbf{Expiration}: Automatic token expiry for security
    \item \textbf{HTTP-only cookies}: Tokens stored in secure, HTTP-only cookies
\end{itemize}

JWT implementation in BondHealth:

\begin{lstlisting}[language=JavaScript, caption=JWT Implementation in BondHealth, basicstyle=\ttfamily\small, breaklines=true]
// Generate JWT token
const generateToken = (user) => {
    return jwt.sign(
        { 
            id: user.user_id, 
            username: user.username, 
            role: user.role 
        }, 
        process.env.JWT_SECRET,
        { expiresIn: '7d' }
    );
};

// Verify JWT token middleware
const verifyToken = (req, res, next) => {
    const token = req.cookies.token;
    
    if (!token) {
        return res.status(401).json({ error: 'No token provided' });
    }
    
    try {
        const decoded = jwt.verify(token, process.env.JWT_SECRET);
        req.user = decoded;
        next();
    } catch (error) {
        return res.status(403).json({ error: 'Invalid or expired token' });
    }
};
\end{lstlisting}

\subsection{bcryptjs}
bcryptjs v2.4.3 provides password hashing for secure credential storage:

\begin{itemize}
    \item \textbf{Salting}: Random salts prevent rainbow table attacks
    \item \textbf{Adaptive}: Work factor can be increased over time
    \item \textbf{Secure comparison}: Timing-safe comparison functions
\end{itemize}

\begin{lstlisting}[language=JavaScript, caption=Password Hashing in BondHealth, basicstyle=\ttfamily\small, breaklines=true]
// Hash password during registration
const saltRounds = 10;

\end{lstlisting}
