\chapter{Testing}

\section{Introduction}
Testing is a critical phase in the software development lifecycle that ensures the reliability, functionality, and quality of the BondHealth system. This chapter discusses the various testing methodologies, test cases, and results obtained during the testing phase. The system was tested across four user roles: patients, doctors, hospital administrators, and lab technicians, with over 50 test cases covering all major functionalities.

\section{Testing Objectives}
The primary objectives of testing BondHealth include:

\begin{itemize}
    \item Verify that all functional requirements are implemented correctly for all four user roles
    \item Ensure the system handles errors gracefully with proper validation
    \item Validate data integrity and security across 15+ database tables
    \item Test system performance under various load conditions
    \item Ensure cross-browser and responsive design compatibility
    \item Verify JWT authentication and role-based access control
    \item Test file upload functionality for reports, certificates, and documents
\end{itemize}

\section{Testing Levels}

\subsection{Unit Testing}
Unit testing involves testing individual components or functions in isolation. For BondHealth, unit tests were written for:

\begin{itemize}
    \item \textbf{Database queries}: Verify correct data retrieval and manipulation for appointments, reports, and prescriptions
    \item \textbf{Authentication functions}: Test password hashing with bcrypt (10 salt rounds) and JWT generation/verification
    \item \textbf{Input validation}: Ensure proper validation of user inputs in registration forms
    \item \textbf{Helper functions}: Test utility functions like date formatting and age calculation
    \item \textbf{Token generation}: Verify JWT tokens contain correct user role and expiration
\end{itemize}

Example unit test for authentication from the codebase:

\begin{lstlisting}[language=JavaScript, caption=Unit Test Example for Authentication, basicstyle=\ttfamily\small, breaklines=true]
// Test password comparison (simulated)
const bcrypt = require('bcryptjs');

test('Password should match hash', async () => {
  const password = 'password123';
  const salt = await bcrypt.genSalt(10);
  const hash = await bcrypt.hash(password, salt);
  const isValid = await bcrypt.compare(password, hash);
  expect(isValid).toBe(true);
});

test('Wrong password should not match', async () => {
  const password = 'password123';
  const wrongPassword = 'wrongpassword';
  const salt = await bcrypt.genSalt(10);
  const hash = await bcrypt.hash(password, salt);
  const isValid = await bcrypt.compare(wrongPassword, hash);
  expect(isValid).toBe(false);
});

// Test JWT token generation
const jwt = require('jsonwebtoken');
const generateToken = (user) => {
  return jwt.sign(
    { id: user.user_id, username: user.username, role: user.role },
    process.env.JWT_SECRET || 'test_secret',
    { expiresIn: '7d' }
  );
};

test('JWT token should contain user role', () => {
  const user = { user_id: '123', username: 'testuser', role: 'patient' };
  const token = generateToken(user);
  const decoded = jwt.verify(token, 'test_secret');
  expect(decoded.role).toBe('patient');
  expect(decoded.id).toBe('123');
});
\end{lstlisting}

\subsection{Integration Testing}
Integration testing verifies that different modules work together correctly. Key integration tests included:

\begin{itemize}
    \item \textbf{User authentication flow}: Login $\rightarrow$ Session creation $\rightarrow$ Protected route access across all dashboards
    \item \textbf{Patient appointment booking}: Patient selects doctor $\rightarrow$ Creates appointment $\rightarrow$ Doctor views it in dashboard
    \item \textbf{Lab report workflow}: Lab technician uploads report $\rightarrow$ Doctor reviews and adds findings $\rightarrow$ Patient views results
    \item \textbf{Prescription workflow}: Doctor writes prescription $\rightarrow$ Patient views and requests refill $\rightarrow$ Order placed
    \item \textbf{Hospital registration flow}: Admin registers hospital $\rightarrow$ Adds doctors $\rightarrow$ Doctors appear in patient search
    \item \textbf{Review system}: User submits review $\rightarrow$ Review appears in public list $\rightarrow$ Helpful count updates
    \item \textbf{Database transactions}: Ensure related tables update consistently (e.g., user creation with profile)
\end{itemize}

\subsection{System Testing}
System testing evaluates the complete, integrated system against requirements. This includes:

\begin{itemize}
    \item \textbf{Functional testing}: Verify all features work as specified for all four roles
    \item \textbf{User interface testing}: Ensure UI elements are responsive and intuitive across dashboards
    \item \textbf{Security testing}: Test authentication, authorization, and data protection
    \item \textbf{Performance testing}: Measure response times and resource usage under load
    \item \textbf{File upload testing}: Test document upload for reports, certificates, and profile images
    \item \textbf{Cross-browser testing}: Verify compatibility across Chrome, Firefox, Safari, and Edge
\end{itemize}

\section{Security Testing}

\subsection{Penetration Testing}
Security tests were conducted to identify vulnerabilities:

\begin{itemize}
    \item \textbf{SQL Injection}: Parameterized queries in all database operations prevented injection attacks
    \item \textbf{XSS (Cross-Site Scripting)}: Input sanitization and HTTP-only cookies blocked malicious scripts
    \item \textbf{CSRF (Cross-Site Request Forgery)}: JWT tokens in cookies prevented CSRF attacks
    \item \textbf{Session Hijacking}: HTTP-only, secure cookies with token expiration prevented hijacking
    \item \textbf{Brute Force}: Rate limiting on login attempts (not yet implemented)
    \item \textbf{JWT Tampering}: Signature verification prevents token modification
    \item \textbf{Password Storage}: bcrypt with 10 salt rounds ensures secure password hashing
    \item \textbf{Role-Based Access}: Middleware verification prevents unauthorized access to routes
\end{itemize}

Common bugs encountered and resolved:

\begin{itemize}
    \item \textbf{Date/time formatting issues}: Inconsistent display across different browsers (fixed with standard formatting)
    \item \textbf{Session expiration}: Not redirecting properly when token expired (fixed with middleware check)
    \item \textbf{Form validation}: Edge cases with special characters and empty submissions (fixed with regex patterns)
    \item \textbf{Database connection}: Timeout under high load (fixed with connection pooling)
    \item \textbf{Responsive layout}: Issues on specific mobile devices (fixed with media queries)
    \item \textbf{File upload}: Large files causing timeout (fixed with size limits and progress indicators)
    \item \textbf{JWT verification}: Token not being sent in some API calls (fixed with cookie-parser middleware)
    \item \textbf{Appointment booking}: Double booking possible (fixed with availability check)
    \item \textbf{Prescription saving}: Medications not saving correctly (fixed with array handling)
    \item \textbf{Review submission}: Duplicate reviews (fixed with rate limiting)
\end{itemize}

\begin{itemize}

The BondHealth platform successfully delivers a comprehensive healthcare management solution that meets the needs of patients, doctors, administrators, and lab technicians.
