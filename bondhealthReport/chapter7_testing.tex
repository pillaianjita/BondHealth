\chapter{Testing}

Testing ensures the reliability, functionality, and quality of BondHealth across four user roles. Over 50 comprehensive test cases covered all major functionalities including authentication, patient management, doctor operations, admin functions, and lab workflows.

\section{Testing Levels}

This section describes the different levels of testing performed to validate BondHealth's functionality, from individual components to complete system workflows.

\subsection{Unit Testing}
Tested individual components in isolation:
\begin{itemize}
    \item Database queries for appointments, reports, prescriptions
    \item Authentication functions: password hashing, JWT generation/verification
    \item Input validation in registration forms
    \item Utility functions (date formatting, age calculation)
\end{itemize}

\subsection{Integration Testing}
Verified module interactions through key workflows:
\begin{itemize}
    \item Login $\rightarrow$ Session creation $\rightarrow$ Protected route access
    \item Patient selects doctor $\rightarrow$ Books appointment $\rightarrow$ Doctor views notification
    \item Lab uploads report $\rightarrow$ Doctor reviews $\rightarrow$ Patient accesses
    \item Doctor writes prescription $\rightarrow$ Patient requests refill
    \item Admin registers hospital $\rightarrow$ Adds doctors to system
\end{itemize}

\subsection{System Testing}
Comprehensive testing across all requirements:
\begin{itemize}
    \item Functional testing: All 50+ features for all four roles
    \item UI/UX testing: Responsiveness, intuitiveness, accessibility
    \item Security testing: Authentication, authorization, data protection
    \item Performance testing: Response times and resource usage
    \item Cross-browser testing: Chrome, Firefox, Safari, Edge
    \item Responsive design testing: Desktop, tablet, mobile views
\end{itemize}

\section{Security Testing}

This section outlines the security vulnerabilities tested and the measures implemented to protect the system from common attacks.

Penetration testing identified and mitigated the following vulnerabilities:

\begin{itemize}
    \item \textbf{SQL Injection}: Parameterized queries prevent injection attacks
    \item \textbf{XSS}: Input sanitization and HTTP-only cookies block script attacks
    \item \textbf{CSRF}: JWT tokens in secure cookies prevent cross-site attacks
    \item \textbf{Session Hijacking}: HTTP-only flag prevents client-side token access
    \item \textbf{Password Storage}: bcrypt with 10 salt rounds secures credentials
    \item \textbf{Role-Based Access}: Middleware prevents unauthorized access
    \item \textbf{JWT Tampering}: Signature verification detects token modification
\end{itemize}

The most common bugs resolved during testing included date and time formatting inconsistencies across different browsers, which affected appointment display. Image upload functionality required additional file size validation to prevent server overload. Appointment rescheduling logic conflicts were identified and fixed to ensure proper time slot availability. Report PDF generation encountered encoding issues with special characters in patient names, which were resolved by implementing proper character encoding. Duplicate review submissions were prevented by adding client-side and server-side checks to ensure each patient could submit only one review per doctor.
