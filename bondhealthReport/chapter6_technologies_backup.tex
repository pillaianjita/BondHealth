\begin{itemize}

\begin{itemize}

\section{Backend Technologies}

\subsection{Node.js}
Node.js v18.17.0 (LTS) is a JavaScript runtime built on Chrome's V8 engine. It enables server-side JavaScript execution and provides an event-driven, non-blocking I/O model. In BondHealth, Node.js powers:

\begin{itemize}
    \item \textbf{Main application server}: \texttt{home.js} runs on port 3005, handling all API routes and serving dashboards
    \item \textbf{Authentication server}: JWT token generation and validation in \texttt{signin.js}
    \item \textbf{Modular architecture}: Separate files for each user role (\texttt{Patient.js}, \texttt{Doctor.js}, \texttt{admin.js}, \texttt{labs.js})
    \item \textbf{Database connectivity}: Connection pooling with Neon PostgreSQL
\end{itemize}

Key Node.js features utilized:
\begin{itemize}
    \item \textbf{Event loop}: Handles multiple concurrent connections efficiently
    \item \textbf{Async/await}: Used throughout for database operations and API calls
    \item \textbf{Module system}: CommonJS modules with \texttt{require()} and \texttt{module.exports}
    \item \textbf{File system}: \texttt{fs} module for reading HTML templates from files
\end{itemize}

\subsection{Express.js}
Express.js v4.18.2 is a minimal and flexible Node.js web application framework. It is the core of BondHealth's backend architecture. Key features used:

\begin{itemize}
    \item \textbf{Routing}: Handling HTTP requests for over 30 different endpoints
    \item \textbf{Middleware}: 
        \begin{itemize}
            \item \texttt{express.json()} - Parsing JSON request bodies
            \item \texttt{cookie-parser} - Parsing cookies for JWT authentication
            \item \texttt{express.static()} - Serving static files
            \item Custom authentication middleware (\texttt{authenticate}, \texttt{authorize}, \texttt{requireAuth})
        \end{itemize}
    \item \textbf{Template rendering}: Serving dynamic HTML content through functions
    \item \textbf{Error handling}: Centralized error management with try-catch blocks
\end{itemize}

Example Express.js route handler from \texttt{home.js}:

\begin{lstlisting}[language=JavaScript, caption=Express.js Route Handler for Appointments, basicstyle=\ttfamily\small, breaklines=true]
// Get today's appointments for doctor dashboard
app.get('/api/appointments/today', authenticate, authorize('doctor'), async (req, res) => {
  try {
    const result = await query(
      `SELECT a.*, p.full_name as patient_name 
       FROM appointments a
       JOIN patients p ON a.patient_id = p.patient_id
       JOIN doctors d ON a.doctor_id = d.doctor_id
       WHERE d.user_id = $1 AND a.appointment_date = CURRENT_DATE
       ORDER BY a.appointment_time`,
      [req.user.id]
    );
    res.json(result.rows);
  } catch (error) {
    console.error('Error fetching appointments:', error);
    res.status(500).json({ error: 'Failed to fetch appointments' });
  }
});

// Book new appointment
app.post('/api/appointments', authenticate, authorize('patient'), async (req, res) => {
  try {
    const { doctor_id, appointment_date, appointment_time, reason, type, location } = req.body;
    
    const patientResult = await query(
      'SELECT patient_id FROM patients WHERE user_id = $1',
      [req.user.id]
    );
    
    const result = await query(
      `INSERT INTO appointments (
        patient_id, doctor_id, hospital_id, appointment_date,
        appointment_time, reason, type, location, status
      ) VALUES (
        $1, $2, (SELECT hospital_id FROM doctors WHERE doctor_id = $2),
        $3, $4, $5, $6, $7, 'pending'
      ) RETURNING *`,
      [patientResult.rows[0].patient_id, doctor_id, appointment_date, 
       appointment_time, reason, type, location]
    );
    
    res.status(201).json(result.rows[0]);
  } catch (error) {
    res.status(500).json({ error: error.message });
  }
});
\end{lstlisting}

\section{Database Technologies}

\subsection{PostgreSQL}
PostgreSQL v15 is a powerful, open-source object-relational database system. BondHealth uses PostgreSQL with the Neon serverless platform. The database schema includes 15+ tables as defined in \texttt{init.sql}:

\begin{itemize}
    \item \textbf{ACID compliance}: Ensures data integrity for medical records and appointments
    \item \textbf{JSON support}: Used for storing medical conditions and allergies as arrays
    \item \textbf{Foreign keys}: Maintain referential integrity between tables (users→patients, doctors→hospitals)
    \item \textbf{UUID support}: Primary keys use UUID for security (e.g., \texttt{uuid\_generate\_v4()})
    \item \textbf{Indexes}: Optimize query performance on frequently accessed columns
    \item \textbf{Transactions}: Group multiple operations into atomic units (e.g., registration with user and profile creation)
\end{itemize}

\subsection{Neon Database Serverless}
Neon is a serverless Postgres platform that separates storage and compute for automatic scaling. It is configured in \texttt{db/config.js}:

\begin{lstlisting}[language=JavaScript, caption=Neon Database Configuration in config.js, basicstyle=\ttfamily\small, breaklines=true]
require('dotenv').config();
const { Pool } = require('@neondatabase/serverless');

const pool = new Pool({
  connectionString: process.env.DATABASE_URL,
  ssl: {
    require: true,
  },
});

const query = async (text, params) => {
  const start = Date.now();
  try {
    const res = await pool.query(text, params);
    const duration = Date.now() - start;
    console.log('Executed query', { text, duration, rows: res.rowCount });
    return res;
  } catch (error) {
    console.error('Query error:', error);
    throw error;
  }
};

const getClient = async () => {
  const client = await pool.connect();
  return client;
};

module.exports = { pool, query, getClient };
\end{lstlisting}

Advantages of Neon used in BondHealth:
\begin{itemize}
    \item \textbf{Serverless architecture}: No infrastructure management, automatic scaling
    \item \textbf{Connection pooling}: Handles multiple concurrent database connections
    \item \textbf{SSL encryption}: Secure connections with \texttt{ssl: { require: true }}
    \item \textbf{Point-in-time recovery}: Data backup and restore capabilities
\end{itemize}

\section{Authentication \& Security Technologies}

\subsection{JSON Web Tokens (JWT)}
JWT (\texttt{jsonwebtoken} v9.0.3) is used for authentication and authorization across all dashboards. Implementation from \texttt{home.js}:

\begin{lstlisting}[language=JavaScript, caption=JWT Implementation in BondHealth, basicstyle=\ttfamily\small, breaklines=true]
const generateToken = (user) => {
    return jwt.sign(
        { 
            id: user.user_id, 
            username: user.username, 
            role: user.role 
        }, 
        process.env.JWT_SECRET || 'fallback_secret', 
        { expiresIn: process.env.JWT_EXPIRE || '7d' }
    );
};

const authenticate = (req, res, next) => {
    let token = req.cookies.token;
    
    if (!token && req.headers.authorization) {
        const authHeader = req.headers.authorization;
        if (authHeader.startsWith('Bearer ')) {
            token = authHeader.substring(7);
        }
    }

    if (!token) {
        return res.status(401).json({ success: false, message: 'Authentication required' });
    }

    const decoded = verifyToken(token);
    if (!decoded) {
        return res.status(401).json({ success: false, message: 'Invalid or expired token' });
    }

    req.user = decoded;
    next();
};

const authorize = (...roles) => {
    return (req, res, next) => {
        if (!req.user) {
            return res.status(401).json({ success: false, message: 'Authentication required' });
        }
        if (!roles.includes(req.user.role)) {
            return res.status(403).json({ success: false, message: 'Access denied' });
        }
        next();
    };
};
\end{lstlisting}

Key JWT features:
\begin{itemize}
    \item \textbf{Stateless authentication}: No server-side session storage required
    \item \textbf{Role-based access}: Tokens contain user role (patient, doctor, admin, lab)
    \item \textbf{HTTP-only cookies}: Tokens stored in secure cookies to prevent XSS
    \item \textbf{Expiration}: Automatic token expiry after 7 days
    \item \textbf{Active session tracking}: \texttt{activeSessions} Map maintains valid sessions
\end{itemize}

\subsection{bcryptjs}
bcryptjs v3.0.3 provides password hashing for secure credential storage:

\begin{itemize}
    \item \textbf{Salting}: Random salts (10 rounds) prevent rainbow table attacks
    \item \textbf{Adaptive}: Work factor can be increased over time
    \item \textbf{Secure comparison}: Timing-safe comparison functions prevent timing attacks
\end{itemize}

\begin{itemize}
    \item \textbf{Node.js/Express} provides a fast, event-driven backend for handling API requests

The combination of these technologies has proven effective in delivering a responsive and user-friendly healthcare platform that meets the needs of patients, doctors, administrators, and lab technicians alike.
